# Turning a Deep Q-Learning Maze into a Browser-Based AI Demo

Case StudyTurning a Deep Q-Learning Maze into a Browser-Based AI Demo
Project GoalThe goal was to make a notebook-based reinforcement-learning project easier to view, test, and share. The Python version showed the algorithm from a development perspective, but it required a local machine-learning environment. The finished browser demo keeps the important behavior intact while presenting the system as a polished, interactive application.
Original SystemThe Python implementation used a maze environment class, an experience-replay class, and a Jupyter Notebook to train a pirate agent. The environment defined the grid, open cells, blocked cells, available moves, reward values, terminal win state, and terminal loss state. The experience-replay logic stored previous episodes so the model could learn from earlier state, action, reward, and next-state combinations.The notebook built the training workflow around a deep Q-learning approach. The agent observed the current maze state, selected an action through epsilon-greedy decision-making, received a reward or penalty, stored the experience, and used repeated training samples to improve future decisions.
Implementation HighlightsThe most important logic was the training loop. It reset the maze from valid starting cells, chose actions through a balance of exploration and exploitation, tracked each episode, stored experiences, updated the model, and checked whether the agent could consistently reach the treasure from valid positions. The neural network predicted Q-values for possible actions based on the current maze state, allowing the agent to improve its route choices over time.For the front-end version, I translated that logic into modular JavaScript. The browser demo separates the maze rules, Q-learning agent, and interface rendering so the project remains easy to inspect and update. The interface includes movement controls, keyboard support, reward tracking, training controls, win-rate metrics, path replay, and a live log of training progress.
Design DecisionsStatic deployment: The app uses browser-ready HTML, CSS, and JavaScript so it can run on GitHub Pages without a server.Preserved logic: The maze layout, valid actions, movement restrictions, and reward-based goal remain aligned with the Python version.Visual learning: Metrics and path animation make the agent’s improvement easier to understand than raw notebook output alone.Clean separation: App logic, training logic, styles, and supporting write-ups are split into separate files.No database: Training data is generated during each session and does not need to be saved between visits.
ResultThe final project turns an AI training notebook into an accessible application. A visitor can play through the maze manually, train an agent, watch the learned route, and see how rewards and exploration shape the final path. It demonstrates algorithmic problem-solving, reinforcement-learning concepts, JavaScript architecture, front-end interaction design, and the ability to convert a technical project into something understandable without removing the original logic.
